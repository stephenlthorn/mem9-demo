06 · architecture

One cluster. Two engines. Four jobs.

Every AI workload — agent state, analytics, vector retrieval, and full-text search — served from a single TiDB cluster. No sync glue. No ETL. One consistency model.
06
Agent State
Inference job metadata & control plane
Durable ACID writes. Each agent's session, task queue, and tool call history.
Analytics
Live analytics on inference data
HTAP — same rows, columnar engine (TiFlash). No ETL, no lag.
Retrieval
RAG without a separate vector DB
HNSW vector indexes inside TiDB. VEC_COSINE_DISTANCE() is a SQL function.
Full-Text
Keyword · BM25
Built-in BM25 via tantivy. No Elasticsearch sidecar required.
Compute
TiDB Server — Stateless SQL Compute
Stateless · horizontally scalable · parses, plans, and routes all four workloads · MySQL-compatible wire protocol
TiKV
Transactional truth
Raft-replicated KV store built on RocksDB. Source of truth for every write. Distributed ACID via Percolator 2PC.
  • Raft-replicated Regions (~256 MB each)
  • Source of truth for all writes
  • Percolator-style distributed 2PC
OLTP
TiFlash
Analytics · Vector · FTS
Columnar extension of TiKV. Same data, three extra indexes. Query optimizer chooses row vs columnar automatically.
  • Async columnar replicas from TiKV
  • HNSW approximate nearest-neighbor vector indexes
  • BM25 full-text indexes via tantivy (TiCI)

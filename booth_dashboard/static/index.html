Memory isn't stored. It's maintained.
Five maintenance duties run continuously inside your TiDB cluster. If you're not running them, your memory is decaying into noise — and your agent is confidently citing diagnoses contradicted months ago.
Three ways agent memory fails in production:
💸
Token Debt
Stuffing all memories into every prompt burns your context window. At scale you are paying to remind the agent what it already knew, every single message.
Signal: context window keeps hitting limits
🫥
Context Amnesia
Session ends. Agent forgets everything. The next user gets the same wrong diagnosis the agent gave the last three users. Cross-session learning simply does not exist.
Signal: agents repeat the same mistakes
📉
Memory Decay
Old facts do not expire. Contradicted conclusions sit beside newer truths. The agent confidently cites a diagnosis it corrected six months ago.
Signal: agents hallucinate coherence from stale data
↓ The five maintenance duties below run continuously inside TiDB to prevent all three.
🛡️
Write control
Only confirmed outcomes persist. Working reasoning stays ephemeral.
status = 'committed' required
📚
Deduplication
Cosine distance < 0.15 merges with existing memory. One strong memory, not ten weak ones.
cosine < 0.15 ⇒ merge
⚙️
Reconciliation
New evidence auto-supersedes contradicted conclusions. Stale facts are linked, not lost.
supersedes_id column
📉
Confidence decay
5% monthly decay without reinforcement. Below 0.30 = auto-deprecated.
confidence *= 0.95 / month
🗜️
Compaction
Weekly re-clustering merges drifted memories. Evidence counts consolidated.
CLUSTER BY embedding
All five duties run inside a single TiDB cluster. No external schedulers. No sync jobs. No Kafka, Redis, or Airflow.
